I. Project Foundation and Core Architecture

Component Technology Role Key Architectural Feature
Backend Node.js (Express) / Prisma (MySQL) Central API & Business Logic Monorepo structure designed for maintainability.
Identity Auth0 Centralized Authentication Single source of truth across all platforms.
Billing Engine Chargebee Subscription Management Handles dynamic plan creation and recurring billing for services.
Client Apps React Web (Admin/Owner Portal) / React Native (Member App) Presentation Layers Modular architecture with separate hooks and components.

II. Identity & Security Flows (The Foundation)

Auth Flow Client Middleware/Service Used Status
Web/Admin Login React Web POST /auth/verify-user / verify-user-admin Exchanges Auth0 token for Internal JWT.
Mobile Login React Native POST /auth/verify-member Verifies Auth0 token and grants access.
Access Control All Protected Routes AuthGatekeeper & Role-Based Access Enforces access based on GYM_OWNER, TRAINER, or MERCHANT roles.
Identity Sync Webhook subscriptionService.processWebhook Data Synchronization: Uses customer.email to find the internal user and set chargebeeCustomerId.
